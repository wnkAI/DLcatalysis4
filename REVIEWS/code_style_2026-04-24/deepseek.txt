## Verbose / Non-idiomatic Python

1. **`v4_ultimate.py:147-151` - `_attn_pool_seq` method**
   - Could use `torch.where` instead of `masked_fill` + `any`
   - **Suggested replacement**: 
   ```python
   def _attn_pool_seq(self, feat, mask, pool_head):
       scores = pool_head(feat)
       scores = scores.masked_fill(~mask.bool(), torch.finfo(scores.dtype).min)
       weights = torch.softmax(scores, dim=1)
       weights = weights * mask.any(dim=1, keepdim=True).float()
       return (feat * weights).sum(dim=1)
   ```

2. **`v4_ultimate.py:155-165` - `_encode_enzyme` method**
   - Uses `getattr` with `None` check pattern repeatedly
   - **Suggested replacement**: Create a helper `_safe_getattr(obj, attr, default=None)`

3. **`v4_ultimate.py:263-284` - `_pairwise_ranking` method**
   - Complex nested loops that could be vectorized
   - **Suggested replacement**: Use PyTorch's `torch.cdist` or `torch.pdist` for pairwise comparisons

4. **`v4_ultimate.py:310-319` - `calculate_metrics` method**
   - Manual NaN handling could use `np.nan_to_num`
   - **Suggested replacement**: 
   ```python
   def calculate_metrics(self, y_pred, y_true):
       yp = y_pred.cpu().numpy().flatten()
       yt = y_true.cpu().numpy().flatten()
       pcc = np.corrcoef(yt, yp)[0, 1]
       scc, _ = spearmanr(yt, yp)
       return {
           "R2": r2_score(yt, yp),
           "PCC": np.nan_to_num(pcc),
           "SCC": np.nan_to_num(scc),
           "MSE": mean_squared_error(yt, yp),
           "MAE": mean_absolute_error(yt, yp),
           "RMSE": float(np.sqrt(mean_squared_error(yt, yp)))
       }
   ```

5. **`v4_ultimate.py:357-372` - `configure_optimizers` method**
   - Complex lambda function for LR scheduling
   - **Suggested replacement**: Use `torch.optim.lr_scheduler.CosineAnnealingWarmRestarts` or a simpler approach

## Redundant Safety Logic

1. **`v4_ultimate.py:147-151` - `_attn_pool_seq` method**

   - **Suggested replacement**: Use `~mask.bool()` directly

2. **`v4_ultimate.py:263-284` - `_pairwise_ranking` method**
   - `max(n_pairs, 1)` is defensive but `n_pairs` is always >= 0
