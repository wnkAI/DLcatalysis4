   [data_module.py](E:\AImodel\DLcatalysis4.0\src\util\data_module.py:182) disables dataset-side DRFP, rxn_center, annot, etc., but does not mutate config or inform `V4Ultimate`. The model is built from the original flags in [v4_ultimate.py](E:\AImodel\DLcatalysis4.0\src\model\v4_ultimate.py:73). Example: if `use_rxn_drfp: true` but no `RXN_drfp` field exists, `_encode_rxn` returns zeros, then `head_rxn(zeros)` still contributes a learned branch bias via `y_rxn`. That is a silent active modality with missing data.

3. Pocket data missing is silent.
   If `pocket_path` is absent, the dataset just leaves `self.pockets = None` in [data_module.py](E:\AImodel\DLcatalysis4.0\src\util\data_module.py:163). `v4_pocket` has no `use_pocket` flag, so it silently trains with `y_struct = 0` and `y_int3d = 0`. For `v4_ultimate`, `use_pocket` can stay true while pocket data is absent.

4. Attention all-False masks are not fully safe inside the cross-attention layers.
   Pooling has a `has_valid` guard in [int3d_cross_attn.py](E:\AImodel\DLcatalysis4.0\src\model\int3d_cross_attn.py:155), but the layer attention itself masks logits then softmaxes in [int3d_cross_attn.py](E:\AImodel\DLcatalysis4.0\src\model\int3d_cross_attn.py:112) and [int3d_cross_attn.py](E:\AImodel\DLcatalysis4.0\src\model\int3d_cross_attn.py:127). If all keys are masked, softmax over all `finfo.min` becomes effectively uniform over invalid tokens, not zero attention. Current loader usually avoids all-False by using dummy nodes, but the module is not robust.

5. `v4_ultimate` ignores configured `struct.gate_init_bias`.
   `v4_pocket` reads it correctly, but `v4_ultimate` hard-codes `-2.0` in [v4_ultimate.py](E:\AImodel\DLcatalysis4.0\src\model\v4_ultimate.py:233) and [v4_ultimate.py](E:\AImodel\DLcatalysis4.0\src\model\v4_ultimate.py:247). Current config also uses `-2.0`, so this is latent but real.

Not a bug: the PyG `__inc__` handling for `MOL_graph_xyz`, `POCKET_edge_index`, and `MOL_rxn_center` looks correct in [data_module.py](E:\AImodel\DLcatalysis4.0\src\util\data_module.py:87). `POCKET_edge_index` is offset by pocket node count, molecule edge indices by atom count, and feature tensors/masks are not offset.
