**Concrete Analysis: QVina for 25k Pair-Specific Docking**

---

### 1. QVina Variants Landscape

| Variant | Status | Suitability for 25k pairs |
|---|---|---|
| **QVina2 (CPU)** | Maintained, OpenMP | **Primary candidate** – fastest CPU Vina derivative |
| **QVina-W** | Windows-only, unmaintained | **Reject** – Linux requirement |
| **QVina-GPU** | Abandoned (2018), no CUDA 12 support | **Reject** – will not compile |
| **QVina1** | Deprecated by QVina2 | **Reject** – no benefit |

**QVina2** is the only viable QVina variant. It uses the same scoring function as AutoDock Vina 1.1.2 but with a faster local optimization (Broyden–Fletcher–Goldfarb–Shanno vs. quasi-Newton). No accuracy gain, only speed.

---

### 2. Throughput Estimates (25,000 pairs, A100 + 16 CPU cores)

**Assumptions:**
- Average ligand: 25 heavy atoms, 5 rotatable bonds
- Receptor: ~5000 atoms (typical enzyme)
- Grid box: 25Å³ (covers ~8Å pocket)
- Exhaustiveness = 8 for all CPU-based docking
- GPU docking uses default batch sizes

| Tool | Seconds/pair | Pairs/day | Total days | Notes |
|---|---|---|---|---|
| **QVina2 (CPU, 16 cores)** | 12–18 s | 4,800–7,200 | **3.5–5.2 days** | Single-threaded per pair, 16 parallel jobs |
| **AutoDock Vina 1.2.x (CPU)** | 45–60 s | 1,440–1,920 | **13–17 days** | 3–4× slower than QVina2 |
| **QVina-GPU** | N/A | N/A | **Fails** | No CUDA 12 support, compile error |
| **Vina-GPU-2.1 (A100)** | 0.3–0.5 s | 172,800–288,000 | **0.09–0.14 days** | Batch of 25k in ~3.5 hours |
| **Uni-Dock (ByteDance, A100)** | 0.2–0.4 s | 216,000–432,000 | **0.06–0.12 days** | Fastest, ~2.5 hours |
| **DiffDock (A100)** | 8–12 s | 7,200–10,800 | **2.3–3.5 days** | Inference only, no training |
| **Gnina (A100, CNN scoring)** | 2–4 s | 21,600–43,200 | **0.6–1.2 days** | Uses Vina for pose generation + CNN rescore |

**Key numbers:**
- **QVina2**: ~4 days (acceptable, fits 1-week budget)
- **Vina-GPU-2.1**: ~3 hours (massive overkill but trivial)
- **Uni-Dock**: ~2.5 hours (fastest)
- **DiffDock**: ~3 days (tight but possible)

---

### 3. Pose Quality for Pocket Anchoring

**QVina2 vs. AutoDock Vina:**
- Identical scoring function → identical pose distributions for top-1
- QVina2 may converge to slightly different local minima (BFGS vs. quasi-Newton), but RMSD between top-1 poses is typically <1.5Å for rigid receptors
- **No meaningful pocket quality loss** – pocket defined by 8Å radius, not sub-Ångström precision

**QVina2 vs. Vina-GPU-2.1:**
- Vina-GPU-2.1 uses the same algorithm, GPU parallelization is bit-exact for same random seed
- **Identical poses** if seed is fixed

**Known failure modes:**
- **Metal cofactors**: Vina family ignores metal coordination. If your enzymes have Zn/Mg/Fe in the pocket, poses will be wrong. **Critical**: check if any of your 7,100 enzymes have metal-dependent catalysis. If >5%, QVina2 fails.
- **Flexible ligands**: QVina2 handles rotatable bonds up to ~15. Beyond that, sampling degrades. Your average (5 rotatable) is fine.
- **Top-1 stability**: With exhaustiveness=8, top-1 pose RMSD between runs (different seeds) is <2Å for 95% of cases. Sufficient for 32-residue pocket definition.

**Verdict**: QVina2 is adequate for pocket anchoring **if** no metal cofactors. If metals present, use Vina-GPU-2.1 with metal-parameterized force field (requires custom AD4 parameter files).

---

### 4. Reviewer-Attack Risk

**Risk of using QVina2 (classical):**
- **Low risk** – reviewers expect classical docking for pocket definition
- **Higher risk** if you claim "novel pair-specific docking" but use a 10-year-old tool. Mitigate: state clearly that docking is only for spatial anchor, not affinity prediction.
- **No risk** of reproducibility complaints – QVina2 is deterministic with fixed seed, open-source, no model weights

**Risk of using DiffDock/Gnina (ML):**
- **Higher risk** – DiffDock is non-deterministic (stochastic sampling), reviewers cannot reproduce exact poses
- Gnina is deterministic but requires CNN model weights (version-dependent)
- **Benefit**: DiffDock handles metal cofactors better (trained on PDBbind with metals)
- **Cost**: 3 days vs. 4 days – negligible difference

**Positioning advice**: Classical docking is **safer** for a regression paper where docking is not the main contribution. ML docking invites "why not use the latest model?" questions.

### 5. Practical Install Pain

| Tool | Build on Linux + CUDA 12 + Python 3.10 | Ligand prep needed | Known issues |
|---|---|---|---|
| **QVina2** | **Clean** – single binary, no dependencies | Yes – Open Babel or Meeko for PDBQT | None |
| **AutoDock Vina 1.2.x** | **Clean** – conda installable | Yes | Slow |
| **Vina-GPU-2.1** | **Moderate** – requires CUDA 11.8 (not 12), may need nvcc workaround | Yes | CUDA 12 compatibility patch exists but untested |
| **Uni-Dock** | **Moderate** – requires CUDA 11.x, Python 3.8–3.10 | Yes | Conda-forge package available, but CUDA 12 support experimental |
| **DiffDock** | **Painful** – requires PyTorch 2.0, DGL, custom CUDA kernels | No – uses RDKit for ligand prep | Frequent version conflicts with conda-forge; needs 30+ GB VRAM for batch inference |
| **Gnina** | **Moderate** – requires Caffe, Boost, CUDA 11.x | Yes | Build system is fragile; prebuilt binaries for Linux available |

**Ligand prep**: All Vina-family tools need PDBQT. Use **Meeko** (Python, pip-installable) – faster than Open Babel, handles protonation states. Add 0.1 s/pair overhead.

---

### 6. Concrete Recommendation

**Primary**: **QVina2 (CPU, exhaustiveness=8, 16 parallel jobs)**
- Expected runtime: **4.2 days** (25,000 pairs × 15 s/pair ÷ 16 cores ÷ 86,400 s/day)
- Deterministic: fix seed=42, get identical poses
- Install: `wget https://github.com/QVina/qvina2/releases/download/v2.1/qvina2.1_linux_x86_64` – done

**Fallback**: **Vina-GPU-2.1** (if QVina2 fails due to metal cofactors or runtime >5 days)
- Expected runtime: **3.5 hours** on A100
- Install: compile from source with CUDA 11.8 compatibility flag
- Deterministic: same seed → same poses as QVina2

**Kill-switch**: If QVina2 exceeds **6 days** (i.e., average >21 s/pair due to larger ligands), switch to Vina-GPU-2.1. If Vina-GPU-2.1 fails to compile (CUDA 12 incompatibility), use **Uni-Dock** (conda installable, 2.5 hours).

**Do not use**: DiffDock (non-deterministic, 3 days, install pain), Gnina (CNN rescore unnecessary for pocket anchoring), AutoDock Vina 1.2.x (13–17 days).

**Final verdict**: QVina2 is the correct tool. It is fast enough, deterministic, and reviewers will not penalize classical docking for a pocket-anchoring task. The 4-day runtime fits your 1-week budget with 40% margin.
